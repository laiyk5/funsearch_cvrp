# 项目进展与后续计划说明

## 一、当前进展

1. **项目主题与目标明确**
   - 选题为“Sample-efficient FunSearch for CVRP”，聚焦于用LLM自动化设计CVRP启发式，并提升采样效率。
   - 已完成项目提案（ICLR格式LaTeX/Word），明确了研究动机、方法路线、评测指标和预期创新点。

2. **代码实现与实验**
   - 已实现CVRP问题的核心数据结构、合成实例生成、经典启发式（Nearest Neighbor、Clarke-Wright Savings）、2-opt局部搜索。
   - 实现了“参数化贪心启发式+进化搜索+早期剪枝+重复检测”的FunSearch原型。
   - 支持完整实验流程，自动输出JSON结果、Markdown/Word报告。
   - 已补充单元测试，保证主流程可复现、可验证。

3. **报告与文档**
   - 已生成详细的里程碑报告（中英文、Markdown/Word），内容涵盖问题背景、方法、实验、分析、局限与未来工作。
   - README已升级，包含环境、运行、测试、数据格式等说明。

4. **可运行性与可重现性**
   - 所有实验均可一键运行，结果可复现（固定seed）。
   - 代码结构清晰，便于后续扩展。

## 二、存在的主要不足

1. **FunSearch核心“LLM生成/演化程序”未实现**（当前仅为参数搜索，未真正调用LLM生成/变异Python函数）。
2. **功能等价检测方式过于简单**（仅对权重取整，未用程序输出判等）。
3. **未集成CVRPLib真实数据集与强基线（如OR-Tools、LKH）**。
4. **实验规模较小，创新点与课程要求的“novelty”尚有差距**。

## 三、后续工作计划

1. **接入LLM生成/演化启发式代码**（最高优先级）
   - 封装OpenAI API或本地大模型接口，实现FunSearch主循环中“LLM生成/变异Python启发式”。
   - 支持自动插入/替换启发式函数，动态评测。

2. **功能等价检测升级**
   - 设计“输入输出向量哈希”或“行为签名”机制，判定两个启发式在同一输入集下输出是否一致，提升重复检测的功能性。

3. **集成CVRPLib标准数据集与强基线**
   - 下载并解析Uchoa等公开CVRP实例，扩展评测范围。
   - 集成Google OR-Tools、LKH等业界强基线，提升对比说服力。

4. **实验与报告完善**
   - 扩大实验规模，补充消融实验与多目标分析。
   - 完善最终报告，突出创新点、实际效果与不足反思。

5. **可选：多目标/多样性/可解释性扩展**
   - 探索多目标FunSearch、novelty search、启发式可解释性等方向，争取更高创新分。
